- **Learnings:**
  - Final-gate equivalence for a multi-row toggle screen is BEST evidenced
    by pairing (a) full-screen pixel-% (the spec's literal threshold) with
    (b) per-region pixel-equality on the unambiguous fixed-content elements
    (here: title and both pill bboxes, all 0-diff). The full-screen number
    answers "does this pass the spec's <1% rule?" and the per-region zeros
    answer "is the residue concentrated in a structural failure or is it
    glyph-aliasing scatter?" — together they distinguish "passes by accident"
    from "passes by construction".
  - Two-row toggle screens have a higher full-screen pixel-diff floor
    (~0.32%) than single-row (audio: ~0.13% pixel — derived from 1035
    bytes / 0.34 bytes/px-channel). The delta tracks glyph-count: Display
    has 14 + 14 + 13 + 4 + 6 = 51 glyphs across labels (Fullscreen/Smooth
    Scaling/Display Options/Save/Cancel), Audio has ~24 glyphs. Each glyph
    contributes ~5–10 anti-aliased edge pixels of palette-index drift —
    the floor scales near-linearly with glyph count, exactly matching the
    main 0.23% / options 0.19% / audio 0.11% / display 0.24% (bytewise)
    progression. Future multi-row screens should expect a floor near
    `0.05% + 0.005% × glyph_count` and not chase residue below that.
  - The "spec gate is full-screen pixel-%, not bytewise residue" distinction
    matters: the OPTIONS Ralph's 0.19% bytewise had pixel-% ≈ 0.06%; the
    DISPLAY Ralph's 0.24% bytewise has pixel-% ≈ 0.32%. The translation is
    not 1:1 because palette-index drift on a single pixel causes 1–3 byte
    deltas (only 1 byte if only one channel changes). Reporting both
    figures keeps the audit honest and the gate measurable against the
    correct denominator.
  - Free-flip cascade tally for the four-Ralph series so far (main, options,
    audio, display): main 1 substantive + 4 free, options 1 + 4 free,
    audio 2 + 3 free (the on-pill A3 fix added one), display 2 + 2 free
    (D0 build harness + D1 composition body, both substantive). Pattern
    holds: substantive-code commits = (1 build harness extension) +
    (1 per *new* sprite-identity ambiguity). DISPLAY had no sprite-identity
    surprises because the prior Audio Ralph had already corrected the
    12/15 atlas; future toggle screens should be similarly cheap as long
    as they reuse the validated 2x2 atlas and the B220x33 widget template.
