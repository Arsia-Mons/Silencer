- **Learnings:**
  - **Toggle widget is the simplest possible widget render**: one
    `BlitSprite` per toggle, no chrome compositing, no label
    overlay, no per-state hover animation at dump time. The codec
    path is identical to button chromes — anchor convention
    applies, sprite.offset baked into BlitSprite. The only widget-
    specific concept is the +42px x-stride which is purely a
    spec-derived constant.
  - **Empty-data overlay slots can be elided** when the runtime
    content is empty on the canonical dump. DrawText with an empty
    string is a no-op anyway, and adding the call wastes source
    bytes. The structural gate is the *position* of the overlay,
    not its glyph rendering — so an inline comment block at the
    slot's would-be location preserves the spec mapping for future
    runtime hookup without producing any framebuffer change.
  - **Bbox-restricted diff metrics are essential** for sub-
    interface gating. Whole-screen diff barely moves with this
    iter (3.06% → 3.17%, which actually went *up* slightly because
    the new sprites paint pixels that disagreed with the
    reference's dim-tint version of the same sprites). The CharInt
    bbox-restricted diff (9.23%) honestly reflects the
    sub-interface state: structural slots filled, runtime tint
    treatment pending. Sub-interface Ralphs should always report
    bbox-local diffs alongside whole-screen, since whole-screen
    metrics dilute per-bbox progress.
