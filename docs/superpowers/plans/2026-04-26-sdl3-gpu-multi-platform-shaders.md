# SDL3 GPU Multi-Platform Shaders — Execution Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the SDL3 GPU backend work on Windows (D3D12 via DXIL) in addition to macOS (Metal via MSL), unblocking the immediate-exit bug at `clients/silencer/src/game.cpp:204`.

**Architecture:** Author each shader in HLSL once. Compile HLSL → DXIL at build time with `dxc` (shipped in the Windows 10/11 SDK). Emit C-array headers via `dxc -Fh -Vn`. Keep MSL strings as runtime-compiled source (Metal accepts MSL source). At init, pass `DXIL|MSL` to `SDL_CreateGPUDevice` and query `SDL_GetGPUShaderFormats(device)` to pick the matching bytecode/source per pipeline.

**SPIRV / Linux is intentionally deferred.** The HLSL sources already carry `[[vk::binding]]` annotations so adding SPIRV later is a CMake plumbing change plus a third arm in the runtime selector — no shader rewrites. Reason for deferral: the Windows SDK's `dxc` lacks SPIRV codegen (built without `ENABLE_SPIRV_CODEGEN`), and a SPIRV-capable DXC requires either the Vulkan SDK or a standalone DirectXShaderCompiler release — neither is on the dev box right now and Linux isn't on the immediate platform list.

**Tech Stack:** C++14, SDL3 GPU (`SDL_gpu.h`), CMake 3.16+, DirectX Shader Compiler (`dxc.exe`).

---

## Constraints

- **HLSL bindings must follow SDL3 conventions** (see `SDL_gpu.h:2590` and `:2695`):
  - Vertex graphics: `t/s` in `space0`, `b` in `space1`. SPIRV set 0 / set 1.
  - Pixel graphics: `t/s` in `space2`, `b` in `space3`. SPIRV set 2 / set 3.
  - Compute: `t` (read-only) in `space0`, `u` (read-write) in `space1`, `b` in `space2`. SPIRV set 0/1/2.
- **Vertex semantics for HLSL:** non-system-value semantics use `TEXCOORD0`, `TEXCOORD1`, … per SDL3 spec (`SDL_gpu.h:2740`).
- **Generated bytecode headers are committed** to the repo. CMake regenerates only when source HLSL changes AND `dxc` is on PATH; otherwise it uses the committed `.h` files. This keeps macOS/Linux builds frictionless without forcing every dev to install DXC.
- **No backwards-compat shims.** Once multi-format works, remove the MSL-only `LoadShader` signature; replace with a `ShaderBundle` struct.
- **Build before commit:** Windows build + smoke launch must pass before opening the PR. Mac verification is out-of-band — call it out in the PR description.

## Task 1 — HLSL ports of all 7 shaders

**Files:** seven `clients/silencer/shaders/*.hlsl` per the file structure above.

For each shader, translate the MSL in `sdl3gpubackend.cpp` faithfully, preserving entry-point names (`vert_screen`, `frag_remap`, `frag_upscale`, `frag_light`, `vert_particle`, `frag_particle`, `update_particles`) and resource indices.

Binding layout (one rule per resource type per stage):

| Stage    | Resource              | HLSL                          | SPIRV (vk::binding) |
|----------|-----------------------|-------------------------------|---------------------|
| vertex   | sampled tex           | `register(t<n>, space0)`      | `(n, 0)`            |
| vertex   | sampler               | `register(s<n>, space0)`      | `(n, 0)`            |
| vertex   | storage buffer (RO)   | `register(t<m>, space0)`      | `(m, 0)`            |
| vertex   | uniform buffer        | `register(b<n>, space1)`      | `(n, 1)`            |
| pixel    | sampled tex           | `register(t<n>, space2)`      | `(n, 2)`            |
| pixel    | sampler               | `register(s<n>, space2)`      | `(n, 2)`            |
| pixel    | uniform buffer        | `register(b<n>, space3)`      | `(n, 3)`            |
| compute  | storage buf RO        | `register(t<n>, space0)`      | `(n, 0)`            |
| compute  | storage buf RW        | `register(u<n>, space1)`      | `(n, 1)`            |
| compute  | uniform               | `register(b<n>, space2)`      | `(n, 2)`            |

For each resource, annotate both `register(...)` (DXIL) and `[[vk::binding(N, set)]]` (SPIRV). DXC honors both.

Key MSL-→-HLSL gotchas:
- Metal's `[[vertex_id]]` → `SV_VertexID`.
- `[[stage_in]]` → struct as input parameter; tag fields with `SV_Position` and `TEXCOORD0`/`TEXCOORD1`/…
- `[[thread_position_in_grid]]` → `SV_DispatchThreadID`.
- Fragment outputs: return type with `SV_Target`.
- Compute thread group: `[numthreads(64,1,1)]` attribute on entry.
- `texture2d<float>::sample()` → `Texture2D<float4>::Sample(SamplerState, float2)`.
- `device const Particle*` (Metal SSBO) → `StructuredBuffer<Particle>` (read-only) in vertex; `RWStructuredBuffer<Particle>` in compute.
- `constant float4&` (Metal UBO) → `cbuffer { float4 …; }`.

- [ ] **Step 1.1 — vert_screen.hlsl** (no resources). Outputs `float4 pos : SV_Position; float2 uv : TEXCOORD0;`. Identical math to the MSL.
- [ ] **Step 1.2 — frag_remap.hlsl**. Pixel stage. `t0,space2` `frame`, `t1,space2` `palette`, `s0,space2` `samp`. Both vk::binding into set 2.
- [ ] **Step 1.3 — frag_upscale.hlsl**. Pixel stage. `t0,space2` `scene`, `s0,space2` `samp`.
- [ ] **Step 1.4 — frag_light.hlsl**. Pixel stage. cbuffer at `b0,space3` matching `LightParams` layout (12 floats).
- [ ] **Step 1.5 — vert_particle.hlsl**. Vertex stage. `StructuredBuffer<Particle>` at `t0,space0`. cbuffer `b0,space1` containing `float4 fi`. Same NDC math.
- [ ] **Step 1.6 — frag_particle.hlsl**. Pixel stage. `t0,space2` palette texture, `s0,space2` sampler. Output is palette tap.
- [ ] **Step 1.7 — comp_particle.hlsl**. Compute. `RWStructuredBuffer<Particle>` at `u0,space1`. cbuffer at `b0,space2` containing `float dt; float3 pad;` (cbuffers are vec4-aligned). `[numthreads(64,1,1)]`.
- [ ] **Step 1.8 — Sanity-compile each HLSL** with `dxc -T <profile> -E <entry> -Fo NUL` and fix any parse errors.

  Also probe `C:/Program Files (x86)/Windows Kits/10/bin/*/x64/dxc.exe` via `file(GLOB)` as a fallback. If not found, emit a `STATUS` message saying generated headers will be used as-is and skip the custom commands.

- [ ] **Step 2.2 — `silencer_compile_shader(<name> <stage> <entry>)` function.**

  Stage maps to a profile: `vertex→vs_6_0`, `pixel→ps_6_0`, `compute→cs_6_0`.

  For each shader, emit two `add_custom_command(OUTPUT …)` calls — one for DXIL, one for SPIRV:
  - DXIL: `${SILENCER_DXC} -T <profile> -E <entry> -Fh <out>.dxil.h -Vn k<Name>DXIL <src>`
  - SPIRV: `${SILENCER_DXC} -T <profile> -E <entry> -spirv -fvk-use-dx-layout -Fh <out>.spirv.h -Vn k<Name>SPIRV <src>`

  Add the outputs as a target source so CMake tracks them. Place them in `clients/silencer/shaders/generated/` (a path that's git-tracked).

- [ ] **Step 2.3 — Wire into top-level `CMakeLists.txt`.**

  `include(cmake/CompileShaders.cmake)` and call `silencer_compile_shader()` for each of the 7 shaders. Add `shaders/generated/` to the silencer target's include directories. Generated headers must be ordering dependencies of the silencer target so they're produced before `sdl3gpubackend.cpp` compiles.

## Task 3 — Refactor backend for multi-format

**Files:**
- Modify: `clients/silencer/src/sdl3gpubackend.cpp`
- Maybe modify: `clients/silencer/src/sdl3gpubackend.h`

- [ ] **Step 3.1 — Replace MSL-only string globals with bundles.** Keep the existing `kVertScreenMSL` etc. strings (Metal still uses MSL source). Add `#include` for each generated `.dxil.h` / `.spirv.h`. Define a `ShaderBundle` struct:

  ```cpp
  struct ShaderBundle {
      const char *msl_src;        // null-terminated MSL
      const Uint8 *dxil; size_t dxil_size;
      const Uint8 *spirv; size_t spirv_size;
      const char *entrypoint;
  };
  ```

  Build one `static const ShaderBundle k<Name>` per shader, populated from generated arrays (`kFragRemapDXIL`, `kFragRemapSPIRV`) and the existing MSL string. The DXIL/SPIRV array sizes are emitted by `dxc -Fh` as `extern unsigned char k<Name>DXIL[]; extern int k<Name>DXILSize;` — verify the exact symbol names after the first generation and adapt the bundle population.

- [ ] **Step 3.2 — `Init()` requests all formats and records the chosen one.** Add `SDL_GPUShaderFormat chosen_format` to the backend. Then:

  ```cpp
  SDL_GPUShaderFormat formats =
      SDL_GPU_SHADERFORMAT_DXIL |
      SDL_GPU_SHADERFORMAT_SPIRV |
      SDL_GPU_SHADERFORMAT_MSL;
  device = SDL_CreateGPUDevice(formats, false, NULL);
  if (!device) { … }
  chosen_format = SDL_GetGPUShaderFormats(device);
  // Reduce to a single format we know how to feed:
  if      (chosen_format & SDL_GPU_SHADERFORMAT_DXIL)  chosen_format = SDL_GPU_SHADERFORMAT_DXIL;
  else if (chosen_format & SDL_GPU_SHADERFORMAT_MSL)   chosen_format = SDL_GPU_SHADERFORMAT_MSL;
  else if (chosen_format & SDL_GPU_SHADERFORMAT_SPIRV) chosen_format = SDL_GPU_SHADERFORMAT_SPIRV;
  else { SDL_Log("no compatible shader format"); return false; }
  ```

  Log the chosen format for triage.

- [ ] **Step 3.3 — Refactor `LoadShader()` to take a `ShaderBundle`.**

  ```cpp
  SDL_GPUShader *LoadShader(SDL_GPUShaderStage stage,
                            const ShaderBundle &b,
                            Uint32 num_samplers,
                            Uint32 num_uniform_buffers = 0,
                            Uint32 num_storage_buffers = 0);
  ```

  Inside, fill `SDL_GPUShaderCreateInfo`:
  - `format = chosen_format`
  - if `chosen_format == MSL`: `code = b.msl_src`, `code_size = strlen(b.msl_src)`
  - if `DXIL`: `code = b.dxil`, `code_size = b.dxil_size`
  - if `SPIRV`: `code = b.spirv`, `code_size = b.spirv_size`
  - `entrypoint = b.entrypoint`
  - bindings as before

  Update every `LoadShader(...)` callsite in `CreatePipelines`, `CreateLightPipeline`, `CreateParticlePipelines` to pass the new bundle constants.

- [ ] **Step 3.4 — Same treatment for the compute pipeline at line 337.** Replace the inline MSL-only `SDL_GPUComputePipelineCreateInfo` with a switch on `chosen_format` selecting `kCompParticle.{msl_src|dxil|spirv}` and `code_size` accordingly. Format goes into `ci.format`.
