Silencer Design System

Static HTML/CSS hydration of the components and screens described in docs/design-system.md. All canvases render at the 640×480 logical resolution the game uses internally; behaviors that depend on JS (button activation animation, caret blink, focus tab order) are wired up with vanilla JS.

What is faked: bitmap glyphs (.BIN sprite banks aren't decodable in pure HTML, so we use the VT323 Google web font as a near-pixel monospace substitute) and sprite art (agency icons, HUD frame art, button backplates etc. are rendered as colored rectangles labelled SPR <bank>#<idx>).

What is faithful: palette hex values from §Color System, all coordinates and dimensions in 640×480 logical pixels, font advances, line heights, hit-test boxes, the button state machine, the caret blink rate, the modal-dialog stacking model, focus tab order.

Foundations

  • Color palette — all 7 ramps and the semantic colors.
  • Typography — font banks 132–136 with their documented advances.

Reusable widgets

  • Button — all 7 variants with hover state machine.
  • Toggle — agency-icon and checkbox modes, radio groups.
  • TextInput — caret blink, scrolling, password mask.
  • TextBox — multi-line auto-scrolling read-only text.
  • SelectBox — single-select list with palette-180 highlight.
  • ScrollBar — up/down arrows with 16px hit zones.
  • Overlay — sprite/text label, animated banks.
  • Interface — focus/tab manager, no separate ring.
  • Horizontal-Stretch Panel — chat-message background (bank 188).
  • Modal Dialog — alert / confirm with optional OK button.
  • Loading Bar — 500×20 px filled rectangle.
  • Drawing Primitives — rect, line, circle, checkered.
