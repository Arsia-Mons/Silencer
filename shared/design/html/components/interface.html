Interface (Container / Focus Manager)

Not a visual component — manages focus, tab order, keyboard routing, modal stacking, and mouse dispatch for child widgets. There is no separate focus ring or border on any widget; focus is expressed entirely through each widget's existing visual states (button hover-brightness, caret blink, etc.).

Tab / Shift-Tab cycling

Tab into the canvas, then Tab through the children. Try Enter on the focused button — it triggers buttonenter behavior. Escape triggers buttonescape ("Cancel"), which is bound below.

Modal stack semantics

One currentinterface at a time. CreateModalDialog() pushes the dialog and saves the previous interface in aftermodalinterface; DestroyModalDialog() restores it. Escape is not bound to dismissal. There is no nested-modal stack — a port that needs nesting must model a stack explicitly.
