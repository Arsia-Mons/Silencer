Button

Seven variants. Hover or keyboard-focus drives the INACTIVE → ACTIVATING(0..4) → ACTIVE → DEACTIVATING(0..4) → INACTIVE state machine. effectbrightness ramps 128 → 136 (about +6% luminance) over 4 ticks (~168 ms). The CSS filter: brightness(1.0625) matches the documented endpoint.

All variants — hover or Tab to see activation

BCHECKBOX — bank 7 idx 18/19
Click any button to see hover state. Tab cycles focus (you can also Shift-Tab back).
State machine (per spec): each ACTIVATING tick → effectbrightness = 128 + (state_i * 2) and res_index = base + state_i.
The 4-tick ramp at 24 Hz = 168 ms. CSS transition matches.
UI sound on enter ACTIVATING: whoom.wav (omitted in this demo).

Hit-testing

Each button uses the sprite-offset bounding box:

x1 = button.x - spriteOffsetX[res_bank][res_index]
x2 = x1 + width
y1 = button.y - spriteOffsetY[res_bank][res_index]
y2 = y1 + height

Hidden buttons (draw = false) are excluded from hit-testing. BCHECKBOX has no hover animation; B156x21 animates brightness only (sprite frame stays at index 24).
