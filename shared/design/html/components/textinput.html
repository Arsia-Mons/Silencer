TextInput

Single-line text field. Caret blinks on a 32-tick (~1344 ms) cycle — visible 0–15, hidden 16–31. Caret is always at the end of the buffer (no arrow-key cursor movement is implemented in the original). Inactive fields render at brightness = 64 (half-dark).

Configured field instances (from §TextInput table)

Login: Username — 180 × 14, bank 133, w 6, max 16
Login: Password — 180 × 14, password mask
Lobby Chat — 360 × 14, max 200 / scroll at 60
Game Create: Name — 210 × 14
Game Create: Min/Max Level / Players / Teams — 20 × 20, numbers only
Inactive (brightness 64)

Caret behavior

  • Caret color: #FCFC00 (palette index 140 — semantic "Caret").
  • Caret height: floor(input.height * 0.8).
  • Caret X: input.x + strlen(visibleText) * fontwidth — always at the trailing edge.
  • Browser caret-color simulates this; the blink cadence is whatever the browser ships (close to 1 Hz, so visually similar).

Intentionally unsupported (do not add)

  • Cursor movement keys (Left, Right, Home, End)
  • Mouse-click caret placement (SetCaretPosition() is broken)
  • Clipboard paste / Ctrl+V
  • IME / composition input
  • Selection / shift-click

Browsers do most of these; in a faithful reimplementation those would all be no-ops.
